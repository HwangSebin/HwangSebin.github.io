Project 설득력 있는 디자인을 하기 위해서는 사용자의 편의성에 대한 이해가 필요합니다. 이를 위해서 어떠한 서비스의 주요 TASK 도출하고, 문제점을 파악하여 IDEATION / 개선점 도출(사용자 NEES도출) 하고 그것을 바탕으로 서비스 설계 및 디자인을 한 프로젝트였으며, UX 및 GUI에 대해 깊숙이 들여다본 프로젝트 였습니다.
services 다달이 정기적으로 나가는 금액에 관해 불편함을 느끼고, 조금의 돈이라도 허투로 쓰고싶지 않은 유저가 사용하는 줄줄 새는 온라인 결제 금액을 한 번에 잡아주는 서비스 LEAKY
User Behavior 정기결제 서비스 결제 전 서비스 종류를 정한다 → 서비스를 저렴하게 쓰고 싶어 이것저것 비교를 한다 → 정기적으로 이걸 쓰게 될까? → 정기적으로 쓴다면 매월 자동결제를 하자 → 한달에 0.99 달러씩 나가는 정도면 괜찮네 → 언제 까지 사용할지 메모장에 적어놓자 → 결제 예정일을 적어놓는다 정기결제 서비스 결제 카드 결제를 한다 → 첫 결제가 난다 → 매월 이 날에 결제가 발생한다는 것을 알게 된다 → 노트나 캘린더로 정기결제 할부 시작일을 기록해둔다 정기결제 서비스 사용 중 일주일 정도 열심히 사용하다 점점 서비스 접속이 뜸해진다 → 결제 내역이 오지만 어떤 내역인지 확인 하고 딱히 행동을 취하지 않는다 → 해당 서비스의 실시간 사용시간을 생각해본다 → 서비스를 다시 열심히 사용해본다 → 해지를 해볼까 생각한다 → 해지를 하려다 포기한다 → 망각 후 다시 결제가 일어난다

Login 로그인을 하지않아도 서비스 사용이 가능하지만 로그인을 한다면 계좌 연동을 할 수 있다.
Setting 계좌 연결, 알림 설정을 손쉽게 사용할 수 있도록 제작하였다.
List View 정기결제의 D-day를 관리하는 메인 대시보드로리스트 형식은 D-DAY에 초점을 맞추었다.
Calendar View 캘린더 뷰는 언제 결 제가 되는지에 초점을 맞추었고,결제되는 날에 어떠한 서비스가 결제되는지 정보를 제공하였다.
Explore Subscription 새로운 정기결제 서비스를 선택할 수 있으며,많은 사람들이 사용하는 서비스의 목록을 제공하고 검색 기능을 추가하여 손쉽게 추가 할 수 있도록 하였다
Subscription View 서비스명, 주기 D-Day 등 사용자가 정보를 제공하여 서비스에 대해 직관적으로 전달하도록 하였다.
Monitoring View 주간 / 월간 사용 시간 및 앱 구동 횟수 모니터링 전체 사용 시간 측정 및 서비스 해지 또는 플랜 전환 제안한다.
